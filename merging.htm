Merging
Merging means combining the changes from one branch into another. It's a fundamental operation in Git that allows you to integrate different lines of development and bring work from feature branches back into the main codebase.
Use Cases:
  • Integrating completed features into the main branch
  • Combining bug fixes with the current codebase
  • Synchronizing work from multiple developers
  • Bringing experimental changes into production code
  • Consolidating multiple commits with squash merge
Examples:
  • Switch to the branch you want to merge into (usually main):
    git switch main

  • You can also use:
    git checkout main
    • Git switch and git checkout can be used to switch branches. Git switch is a safer command for branch operations only. Git check is a command with multiple functions, which can lead to confusion and potential loss of data if not used correctly.

  • Update your branch with the latest changes before merging:
    • git pull
    You should ensure that the branch you switched to is updated with the latest changes from the repo before merging.

  • Merge a branch into your current branch:
    • git merge demo1
    This merges the "demo1" branch into your current branch.

  • Squash merge (combine all changes into a single commit):
    • git merge --squash demo1
    Squash merge is used if you want all changes from a specific branch to your current branch in a single commit.

  • Abort a merge:
    • git merge --abort
    This is handy if you have any issues during a merge like a conflict that you don't want to resolve. You can cancel the merge and revert things back to before.
